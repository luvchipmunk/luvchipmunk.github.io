# TCP/IP

ARPANET에서 처음개발된 통신 프로토콜이며,

각각의 네트워크에 접속되는 호스트들은 고유의 주소를 가지고 있어 자신이 속한 네트워크뿐만 아니라 다른 네트워크에 연결되어 있는 호스트까지도 서로 데이터를 주고받을 수 있도록 만들어져 있는 것이 특징.(+ 특정 운영체제에 국한되지 않음)

호스트내 TCP/IP들은 InterNIC란 단체에서 관리/분배되고 있다

- ## IP주소

전 세계 인터넷을 사용하는 모든 사람은 모두 서로 다른 IP주소를 가지고 있다.

다른 사람이 나와 똑같은 IP주소를 쓴다면 IP주소가 충돌이 나기 때문에, 둘 중 하나는 인터넷을 못 쓰게 된다.

IP 주소 고갈로 인해서, 

인터넷으로 나갈 때만 공인 IP 주소를 가지고 나가는 방식인

NAT(Network Address Translation)나 동일한 IP 주소를 가지고 여러 명이 인터넷에 접속하면서 포트 넘버만을 바꾸는 PAT등이 사용된다.

(주로 집에서 쓰는 공유기가 NAT기능을 탑재하고 있다고 봐야함)

IP주소는 이진수로 만들어져있으며, 이진수 8자리는 최대 11111111. 십진수로 변환하면 255가 되는데, 이러한 묶음이 4묶음이 존재하여 IP주소는 32개의 이진수로 만들어진다.

따라서 만들 수 있는 IP 주소는 총 232개이다.
다만, 이진수 묶음(8자리)마다 점을 찍어 구분하는데

32자리를 모두 이진수로 표현하기는 어려워 이진수를 모두 십진수로 표시해서 사용하는게 보편적이다.

cf) NAT vs DHCP

좀 헷갈리는데.. 왜냐하면 공유기 설정 들어가보면 NAT 설정이 있고, DHCP 할당이라는게 눈에 좀 보인다.

애초에 의미가 다른 듯 싶다.

인터넷 검색을 통해 여러가지 참고해보면,

| NAT - 매핑(변환) -> 자신이 사용하는 주소로 보낸 패킷이 그 주소로 매핑되는 주소로 다시 쓰여지는 과정 |
| ------------------------------------------------------------ |
| **할당 - DHCP -> 호스트가 하나의 IP주소를 받아 그 주소를 통해 통신** |

내용을 좀 읽어보면 나눠주는 의미에 가까운게 DHCP.

사설 IP단 느낌으로 나눠준 상황을 다시 공인 IP로 변환시켜 나가는게 NAT.



[KTWORD](http://www.ktword.co.kr/test/view/view.php?no=1676) 에서 설명하는 내용도 참조하면 좋을 것 같다.

.. 뒤로 넘겨보니 교재 부가적인 설명란에 DHCP에 대한 부가적인 설명이 존재했다

TCP/IP의 가장 큰 특징은 IP주소가 겹치지 않는 것.

다만, 동일한 네트워크 부분을 가져야하며, 서브넷 마스크는 동일하게 가져가야 한다고 한다.

IP주소단을 겹치지 하지 않기위해 DHCP를 쓰는데,

DHCP는 IP 주소를 자동으로 배정해준다. 때문에, 관리자가 IP주소를 직접 하나하나 부여할 필요가 없고, 내부 IP를 할당받아쓰는 유저 역시 본인 IP를 굳이 알 필요는 없다.

결론은 DHCP는 PC 하나하나 IP 주소를 미리 지정해두지 않고, DHCP가 그 네트워크에 필요한 IP주소를 보유하고 있다가 IP 주소를 요구하는 PC에 자동으로 그때그때 분배하는 시스템.

물론 다 쓰고 남은 IP주소는 회수한다.(라우터단에서 내부 IP를 고정부여하는게 아닌 이상)
